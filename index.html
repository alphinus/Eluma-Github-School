    Glossar

    Kurz erklärt, direkt mit Quests verbunden. Theorie nur, wenn sie sofort beim Spielen hilft.

    Commit

    Ein gespeicherter Zwischenstand deiner Änderungen. Wie ein Speicherpunkt in einem Spiel.

    Passende Quest: L1-M1 — Button-Farbe ändern

    Push

    Der Moment, in dem dein lokaler Stand zu GitHub hochgeladen wird. Vorher sieht nur dein Computer die Änderung.

    Passende Quest: L2-M2 — Branch pushen und prüfen

    Branch

    Ein separater Arbeitszweig für eine Aufgabe. So bleibt `main` sauber und stabil.

    Passende Quest: L3-M1 — Idee in eigenem Branch anlegen

    Pull Request (PR)

    Eine Bitte, deinen Branch zu prüfen und zu übernehmen. Bitte reviewen, bevor das auf `main` landet.

    Passende Quest: L1-M1 — Button-Farbe ändern

    Review

    Die Prüfung deines Pull Requests durch die andere Person. Feedback ist Teil des Spiels, nicht der Fehler.

    Passende Quest: L5-M2 — Ersten Review-Kommentar schreiben

    Merge

    Das Zusammenführen einer freigegebenen Änderung. Danach ist dein Fortschritt im Hauptzweig sichtbar.

    Passende Quest: L5-M4 — PR nach Review-Feedback überarbeiten

    Konflikt

    Eine Stelle, an der Git nicht automatisch entscheiden kann, welche Änderung gelten soll. Kein Fehler, sondern eine Boss-Entscheidung.

    Passende Quest: L4-M1 — Überschriften-Konflikt

    Quest Board

    Hier laufen alle Kampagnen zusammen. Nur freigeschaltete Quests sind direkt spielbar, der Rest bleibt sichtbar als Ziel.
