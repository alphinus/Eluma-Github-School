Glossar

Die wichtigsten Git- und GitHub-Begriffe einfach erklärt.

Commit

Ein gespeicherter Zwischenstand deiner Änderungen. Wie ein Speicherpunkt in einem Spiel — du kannst jederzeit dorthin zurückkehren.

Passende Mission: L1-M1 — Button-Farbe ändern

Push

Der Moment, in dem dein lokaler Stand zu GitHub hochgeladen wird. Vorher sieht nur dein Computer die Änderungen — nach dem Push auch GitHub.

Passende Mission: L2-M2 — Branch pushen und prüfen

Branch

Ein separater Arbeitszweig für eine Aufgabe. Stell dir vor, du kopierst ein Blatt Papier und schreibst auf der Kopie weiter — das Original (main) bleibt sauber.

Passende Mission: L3-M1 — Idee in eigenem Branch anlegen

Pull Request (PR)

Eine Bitte, deinen Branch mit main zu vergleichen und zu übernehmen. Du sagst damit: „Schau dir meine Änderungen an, bevor sie auf main landen."

Passende Mission: L1-M1 — Button-Farbe ändern

Review

Die Prüfung deines Pull Requests durch die andere Person. Der Reviewer schaut sich die Änderungen an und gibt Feedback: Approve, Comment oder Request Changes.

Passende Mission: L5-M2 — Ersten Review-Kommentar schreiben

Merge

Das Zusammenführen einer freigegebenen Änderung. Dein Branch wird mit main verschmolzen und die Änderungen sind für alle sichtbar.

Passende Mission: L5-M4 — PR nach Review-Feedback überarbeiten

Konflikt

Eine Stelle, an der Git nicht automatisch entscheiden kann, welche Änderung gelten soll. Passiert wenn zwei Personen die gleiche Zeile ändern. Kein Fehler — eine Entscheidung!

Passende Mission: L4-M1 — Überschriften-Konflikt
