# Lernpfad

## Wie dieses System funktioniert

Du lernst GitHub nicht durch Lesen, sondern durch **Machen**. Jedes Level baut auf dem vorherigen auf. Jede Mission hat ein sichtbares Ergebnis.

## Die 5 Levels

### Level 1 — Die ersten sicheren Schritte

**Was du lernst:** Datei ändern, committen, pushen, PR erstellen.

**Danach kannst du:**
- Eine Datei auf einem Branch ändern
- Einen Commit mit sinnvoller Nachricht erstellen
- Den Branch zu GitHub pushen
- Einen Pull Request öffnen

**Typische Missionen:**
- Button-Farbe ändern
- Überschrift anpassen
- Neue Ideenkarte hinzufügen
- GitHub-App auf dem Handy installieren

---

### Level 2 — Lokal vs. Remote verstehen

**Was du lernst:** Der Unterschied zwischen deinem Computer und GitHub.

**Danach kannst du:**
- Erklären, warum ein Push nötig ist
- Verstehen, warum GitHub nicht automatisch alles weiß
- Änderungen von GitHub auf deinen Computer holen
- Einen PR nach Feedback ergänzen

**Typische Missionen:**
- Lokale Änderung machen, ohne zu pushen — Unterschied erklären
- Branch pushen und im Browser prüfen
- Fremden Merge auf GitHub sehen und lokal synchronisieren

---

### Level 3 — Branches bewusst nutzen

**Was du lernst:** Warum pro Aufgabe ein eigener Branch sinnvoll ist.

**Danach kannst du:**
- Für jede Aufgabe den richtigen Branch-Typ wählen
- Branch-Namen nach dem Namensschema erstellen
- Zwischen Branches wechseln
- Verstehen, dass `main` immer sauber bleibt

**Typische Missionen:**
- Neue Idee in einem `idea/`-Branch anlegen
- UI-Anpassung in einem `feature/`-Branch
- Dokumentationsfix in einem `fix/`-Branch

---

### Level 4 — Konflikte absichtlich erzeugen und lösen

**Was du lernst:** Merge-Konflikte sind kein Fehler, sondern eine Entscheidung.

**Danach kannst du:**
- Verstehen, wie Konflikte entstehen
- Einen Konflikt erkennen und lesen
- Den Konflikt bewusst lösen (nicht blind klicken)
- Dokumentieren, welche Version übernommen wurde und warum

**Typische Missionen:**
- Beide ändern dieselbe Überschrift
- Beide ändern denselben CSV-Eintrag
- Konflikt lokal öffnen und manuell auflösen

---

### Level 5 — Reviews und Qualität

**Was du lernst:** PRs nicht nur abgeben, sondern prüfen und verbessern.

**Danach kannst du:**
- Einen PR inhaltlich lesen und verstehen
- Einen Review-Kommentar schreiben
- „Request Changes" sinnvoll einsetzen
- Feedback in deinen PR einarbeiten
- Nur bei sauberem Review mergen

**Typische Missionen:**
- PR mit schlechter Beschreibung verbessern
- Review-Kommentar schreiben
- PR nach Feedback überarbeiten

---

## Fortschritt

Dein Fortschritt wird in der App angezeigt. Jede abgeschlossene Mission wird als „done" markiert, wenn der zugehörige PR gemerged wurde.

## Tipps

- Mach eine Mission nach der anderen — nicht überspringen
- Lies die Mission-Beschreibung komplett, bevor du anfängst
- Frag nach, wenn etwas unklar ist
- Fehler sind erlaubt und erwünscht — daraus lernt man am meisten
